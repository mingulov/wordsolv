# Project rename: `wordlesolv` → `wordsolv`

**Date:** 2026-07-20
**Status:** approved

## Why rename

"Wordle" is a registered trademark of The New York Times, which has issued
takedown requests against projects using the mark. A solver *assistant* is a
substantially weaker target than a clone, and referring to Wordle to describe
what the tool works with is ordinary nominative use. But putting the mark in
the project's own name — repo, package names, PWA identity, deploy URL — is
the part that carries avoidable risk, and it is cheapest to fix now: the
project has no git remote, has never been deployed, and has no users.

A secondary reason: the name overstates the constraint. The solver handles
English and Russian, word lengths 4–8, and 1–16 boards. Wordle is one
configuration of many.

## Name

**`wordsolv`** — the existing name minus the trademark.

Selection criteria, in the order they were weighted:

1. **Understandable without explanation.** A reader sees the name and knows
   what the project does. This ruled out the abstract/coined direction
   (`clarus`, `certior`, `entro`, `entrolog`) that an earlier pass had
   favoured: those names are opaque until someone explains them, and acquire
   meaning only through promotion this project will never do.
2. **No trademark in the name.** Satisfied by construction.
3. **Legible to both English and Russian speakers.** "word" + "solv" are
   near-universally parsed by anyone already reading a Latin-script UI.
   `lexsolv` (from lexicon / лексикон) was the runner-up and is more
   distinctive in search results, but "lex" needs a beat of thought that
   "word" does not.
4. **Publishable.** Both the bare npm package `wordsolv` and the scope
   `@wordsolv` were verified to have nothing published under them
   (2026-07-20). Note this does not prove the org is unregistered — that
   needs an authenticated check before any actual publish.

Continuity is a real bonus: the rename is a two-letter deletion, so every
identifier keeps its existing shape and the diff stays reviewable.

### Description

Wordle and Quordle are named only in prose — repo description, README
subtitle, PWA `description` — never in the name, package names, or URLs.

- **EN:** Solver assistant for Wordle, Quordle and other word-grid games —
  English & Russian, 1–16 boards.
- **RU:** Помощник-решатель для Wordle, Quordle и других словесных игр.

## Scope

### Mechanical (global `wordlesolv` → `wordsolv`)

| Surface | Detail |
|---|---|
| Workspace names | root `wordlesolv`; `@wordlesolv/solver-core`; `@wordlesolv/web` |
| Import specifiers | 30 occurrences of `@wordlesolv/solver-core` |
| PWA identity | `name`, `short_name` in `apps/web/vite.config.ts`; `<title>` in `index.html` |
| CI | `npm test -w @wordlesolv/web` in `.github/workflows/deploy-pages.yml` |
| Docs | `README.md`, `CLAUDE.md`, `packages/solver-core/README.md`, `BENCHMARKS.md` |
| Lockfile | regenerated by `npm install`, never hand-edited |

### Data-format and state identifiers

These are not labels; they identify a file format and persisted state.
Decision: **rename all of them, with no migration or dual-read code.**
Nothing has ever been deployed, so there are no users to migrate, and
compatibility shims for zero users are dead weight. The cost is that
sessions in a local dev browser are orphaned — acceptable. The benefit is
that the retired name does not survive inside a file-format magic string.

1. **Dictionary asset header** `#wordlesolv-dict v1` → `#wordsolv-dict v1`.
   Lives in three places that must change atomically: `serializeDict`, the
   `parseDictAsset` regex (both in `packages/solver-core/src/dictionary.ts`),
   and the first line of all ten committed `dict/assets/*.txt`. The assets
   are regenerated with `npx tsx dict/build.ts` rather than edited, and
   `apps/web/public/dict/` follows via `scripts/copy-assets.mjs`. Changing
   the code without regenerating the assets fails every dictionary test.
   The format version stays `v1`: the schema is unchanged, only the vendor
   tag.
2. **localStorage keys** `wordlesolv:{sessions,quarantine,settings}` →
   `wordsolv:*`.
3. **Game-file comment header and export filenames** — the `# wordlesolv
   game file` banner emitted by `gameFileTemplate`/`serializeGameFile`, and
   the `wordlesolv-game.{json,txt}` download names.

### Deliberately out of scope

- **`docs/superpowers/specs/` and `docs/superpowers/plans/`** keep the old
  name. They are dated records of decisions made under it; rewriting them
  would misrepresent history. This document supersedes them on naming.
- **The working-directory name** (`~/src/m/wordlesolv`) is the operator's to
  change; nothing in the repo depends on it.
- **Solver behaviour.** No algorithm, dictionary content, or tuning changes,
  so `openers.json` and the recorded figures in `BENCHMARKS.md` remain valid
  and the 10-minute benchmark gates need not be re-run.

## Verification

1. `npm install` — relink workspaces under the new scope, regenerate lockfile.
2. `npm run typecheck --workspaces` — catches every stale import specifier.
3. `cd packages/solver-core && npx vitest run` — fast suite; covers the
   dictionary header round-trip and game-file format.
4. `npm test -w @wordsolv/web` — covers the localStorage keys.
5. `npm run build -w @wordsolv/web`.
6. `grep -ri wordlesolv` returns hits only under `docs/superpowers/`.

Steps 2–4 are the real gate: a missed import specifier fails typecheck, and a
half-applied dictionary header fails the solver-core suite.
